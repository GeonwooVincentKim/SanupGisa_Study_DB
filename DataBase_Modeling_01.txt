1. 데이터베이스 구조 -> 단순 명료하게 구축

2. 정보시스템의 전반적인 특징
- 장점
  = 1. 다양한 기종의 하드웨어 플랫폼을 가지고 있다 (실행하고 있다)
  = 2. 다양한 데이터베이스를 가지고 있다.
  = 3. 수 많은 데이터 모델이 분할되어 있다.
  = 4. 모든 업무의 설계 중심이 업무 흐름대로 설계가 되어 있다.

- 단점
  = 1. 데이터 중복 및 일관성 문제
  = 2. 이력 관리가 제대로 되어 있지 않다.

3. 데이터 모델
- 1. 데이터 모델 = 건축물의 도면
- 2. 데이터 모델 잘못 된 종류
  = 1. 형제형
     -> 상하에 관계 없이 누구나 같은 위치로 설정됨
     -> 단군 밑에 있는 할아버지, 아버지, 나, 삼촌 모두 같은 위치로 설정되어 있음 (잘못된 예시)
  
  = 2. 족보형
     -> 족보처럼 부계(父系) 중심으로 작성된 데이터 모델
     -> 아버지 피만 흐르는 것이 아닌 어머니의 피도 흐른다
     -> 어느 한쪽으로부터만 가지고 오는 것이 아닌 2개 이상의 상위 데이터 모델로부터 상속을 받아야 한다.

  = 3. 네트워크형
     -> 네트워크 도면처럼 길게 선이 하나 늘어져 있고, 중간중간에 하나씩 라인을 빼서 단말기를 붙이는 형식
     -> 부모가 누구이고, 어떤 피가 흘러 들어왔는지에 대한 정보 모호
