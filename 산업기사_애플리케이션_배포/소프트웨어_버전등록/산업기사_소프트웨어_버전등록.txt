소프트웨어 패키징의 형상관리

<형상 관리>
- 1. 개발과정에서 S/W 의 변경사항을 관리하기 위해 개발된 일련의 활동
- 2. S/W 개발 전체 비용 줄이고, 개발 과정의 여러 방해요인 최소화 보증


형상 관리 기능

- 1. 형상 식별 = 형상관리 대상에 이름 & 관리번호 부여, 계층구조 구분, 수정 추적 용이
- 2. 버전 제어 = 다른 버전 형상항목 관리, 특정 절차와 도구를 결합시키는 작업
- 3. 형상 통제 = 형상항목 변경 요구 검토, 현재의 기준선 반영되도록 조정하는 작업
- 4. 형상 감사 = 기준선의 무결성 평가, 확인, 검증, 검열, 공식적으로 승인하는 작업
- 5. 형상 기록 = 형상 식별, 통제, 감사 작업 결과를 기록, 관리, 보고서 작성하는 작업


소프트웨어의 버전 등록 관련 주요 기능

- 1. 저장소 = 최선 버전 파일들 & 변경 내역에 대한 정보들이 저장되어 있는 곳
- 2. 가져오기 = 버전관리 X, 아무것도 없는 저장소에 처음으로 파일 복사
- 3. 체크아웃 = 프로그램 수정, 저장소에서 파일 받아옴, 소스파일 & 버전관리 파일
- 4. 체크인 = 체크아웃 한 파일의 수정 완료, 저장소 파일을 새로운 버전으로 갱신
- 5. 커밋 = 체크인 수행, 갱신내용 있으면 Conflict 알리고, diff 도구 이용해 수정, 갱신
- 6. 동기화 = 저장소에 있는 최신 버전, 자신의 작업 공간 동기화


소프트웨어 버전 등록 과정

- 1. 가져오기 = 개발자가 저장소에 신규로 파일 추가
- 2. 인출 = 수정 작업 진행할 개발자, 저장소 추가된 파일을 자신의 작업공간으로 인출
- 3. 예치 = 인출한 파일을 수정, 설명을 붙여 저장소에 예치
- 4. 동기화 = 커밋 후, 새로운 개발자 작업공간 동기화, 기존 개발자 추가했던 파일 전달
- 5. 차이 
  = 1. 새로운 개발자 추가된 파일 수정 기록 확인
  = 2. 이전 개발자 처음 추가한 파일과 이후 변경된 파일의 차이 확인
