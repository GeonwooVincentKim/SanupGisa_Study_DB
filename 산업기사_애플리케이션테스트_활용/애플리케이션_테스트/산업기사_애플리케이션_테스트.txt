애플리케이션 테스트
-> 1. 애플리케이션에 잠재되어 있는 결함을 찾아내는 일련의 행위 또는 절차
-> 2. 고객 요구사항 만족하는지 확인, S/W 기능 정확히 수행하는지 검증


애플리케이션 테스트의 기본 원리
-> 1. 완벽한 테스트 불가능 
   = 잠재적인 결함 줄이기 가능, S/W 에 결함 없음 증명 불가

-> 2. 파레토 법칙 = 애플리케이션의 20% 모듈 중 전체 결함의 80% 가 발견
-> 3. 살충제 패러독스 
   = 동일한 테스트 케이스로 동일한 테스트 반복, 더 이상 결함 발견 안됨

-> 4. 테스팅은 정황 = 정황에 따라 테스트를 다르게 수행
-> 5. 오류-부재의 궤변 
   = 결함 모두 제거, 사용자 불만족 - 품질 높다고 말할 수 없음

-> 6. 테스트와 위험은 반비례 = 테스트 많이 진행 - 미래에 발생할 위험 감소
-> 7. 테스트의 점진적 확대 = 테스트 작은 부분부터 점점 확대하며 진행
-> 8. 테스트의 별도 팀 수행 = 테스트 별도 팀에서 수행
