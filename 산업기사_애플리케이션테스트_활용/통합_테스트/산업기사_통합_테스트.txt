통합 테스트

- 1. 단위 테스트 끝난 모듈 통합하는 과정, 발생하는 오류 및 결함 찾는 테스트 기법
- 2. 종류
  -> 1. 비점진적 통합 방식
     = 1. 단계적 통합 X, 모든 모듈이 미리 결합되어 있는 프로그램 전체 테스트
     = 2. 빅뱅 통합 테스트 방식

  -> 2. 점진적 통합 방식
     = 1. 모듈 단위로 단계적 통합
     = 2. 하향식 통합 테스트, 상향식 통합 테스트, 혼합식 통합 테스트


하향식 통합 테스트

- 1. 상위 모듈 -> 하위 모듈 통합하는 방식
- 2. 깊이 우선 통합법, 넓이 우선 통합법
- 3. 절차
  = 1. 주요 제어 모듈은 작성된 프로그램 사용, 종속 모듈들은 스텁 사용
  = 2. 깊이우선, 넓이우선 통합 방식, 하위모듈 스텁들 한 번에 하나씩 실제 모듈로 교체
  = 3. 모듈 통합될 때마다 테스트실시
  = 4. 새로운 오류가 발생하지 않음을 보증하기 위해 회귀 테스트 실시


상향식 통합 테스트

- 1. 하위 모듈 -> 상위 모듈 통합하는 방식
- 2. 절차
  = 1. 하위 모듈들을 클러스터로 결합
  = 2. 상위 모듈, 데이터 입출력 확인, 더미 모듈인 드라이버 작성
  = 3. 통합된 클러스터 단위로 테스트
  = 4. 클러스터는 프로그램 구조 상위로 이동, 드라이버는 실제 모듈로 대체


혼합식 통합 테스트

- 1. 하위 수준에서는 상향식 통합, 상위 수준에서는 하향식 통합, 최적의 테스트 지원
- 2. 샌드위치식 통합 테스트 방법


회귀 테스팅

- 1. 통합 테스트로 인해 변경된 모듈이나 컴포넌트 새로운 오류가 있는지 확인
- 2. 이미 테스트된 프로그램 테스팅 반복
- 3. 수정 모듈, 컴포넌트 영향, 오류 여부 확인, 새로운 오류 발생 X 보증, 반복 테스트
