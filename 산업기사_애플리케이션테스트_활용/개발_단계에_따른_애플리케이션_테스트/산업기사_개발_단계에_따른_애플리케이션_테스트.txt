개발 단계에 따른 애플리케이션 테스트
- 1. 단위, 통합, 시스템, 인수 테스트로 분류, 테스트 레벨
- 2. 애플리케이션 테스트 + S/W 개발 단계 연결 = V-모델


단위 테스트
- 1. 모듈이나 컴포넌트에 초점을 맞춰 테스트
- 2. 구조 기반 테스트, 사용자 요구사항 기반으로 한 기능성 테스트를 최우선
- 3. 인터페이스, 외부적 I/O, 자료 구조, 독립적 기초 경로, 오류 처리 경로, 경계 조건


통합 테스트
- 1. 단위 테스트가 완료된 모듈들 결합, 하나의 시스템으로 완성시키는 과정의 텍스트
- 2. 모듈 간, 통합된 컴포넌트 간 상호 작용 오류 검사


시스템 테스트
- 1. 개발된 S/W 가 완벽하게 수행되는가를 점검하는 테스트
- 2. 기능적 요구사항 & 비기능적 요구사항, 각각 만족하는지 테스트


인수 테스트
- 1. 사용자 요구사항을 충족하는지에 중점을 두고 테스트
- 2. 사용자가 직접 테스트
- 3. 테스트 종류
  -> 1. 사용자 인수 테스트 = 사용자가 시스템 사용 적절성 여부 확인
  -> 2. 운영상 인수 테스트
     = 1. 시스템 관리자가 시스템 인수 시 수행하는 테스트 기법
     = 2. 백업/복원 시스템, 재난 복구, 사용자 관리, 정기 점검

  -> 3. 계약 인수 테스트 = 계약상 인수/검수 조건 준수 여부 확인
  -> 4. 규정 인수 테스트 = S/W 의 정부 지침, 법규, 규정에 맞게 개발
  -> 5. 알파 테스트 
     = 1. 사용자가 개발자 앞에서 행하는 테스트 기법
     = 2. 통제된 환경, 오류와 사용상의 문제점을 사용자 & 개발자 함께 확인 및 기록

  -> 6. 베타 테스트
     = 1. 선정된 최종 사용자가 여러 명의 사용자 앞에서 행하는 테스트 기법
     = 2. 실업무를 가지고 사용자가 직접 테스트
