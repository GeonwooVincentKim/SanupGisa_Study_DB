테스트 케이스 (TC)

- 1. S/W 가 사용자 요구사항 준수 여부 확인, 테스트 항목에 대한 명세서
- 2. ISO/IEC/IEEE 29119-3 표준에 따른 TC 의 구성 요소
  -> 1. 식별자 = 항목 식별자, 일련번호
  -> 2. 테스트 항목 = 테스트 대상 (모듈 or 기능)
  -> 3. 입력 명세 = TC 수행 시 예상되는 출력 결과
  -> 4. 환경 설정 = 필요한 H/W 나 S/W 환경
  -> 5. 특수 절차 요구 = TC 수행 시 특별히 요구되는 절차
  -> 6. 의존성 기술 = TC 간 의존성


테스트 시나리오

- 1. TC 를 적용하는 순서에 따라 여러 개의 TC 를 묶은 집합
- 2. 테스트 순서에 대한 구체적인 절차, 사전 조건, 입력 데이터 설정


테스트 오라클

- 1. 테스트 결과 정확 여부 판단, 사전에 정의된 참 값을 비교하는 기법
- 2. 특징
  = 1. 제한된 검증 -> 모든 TC 에 적용 불가
  = 2. 수학적 기법 -> 테스트 오라클 값의 수학적 기법 이용
  = 3. 자동화 기능 -> 테스트 대상 프로그램 실행, 결과 비교, 커버리지 측정 자동화


테스트 오라클의 종류

- 1. 참 오라클 -> 테스트 오라클 값 기대하는 결과 제공, 발생된 모든 오류 검출 가능
- 2. 샘플링 오라클 -> 특정 몇 TC 입력 값, 기대하는 결과, 전수 테스트 불가능한 경우
- 3. 추정 오라클 -> 특정 TC 입력 값 기대하는 결과, 나머지 입력 값 추정으로 처리
- 4. 일관성 검사 오라클 -> App 변경, TC 수행 전과 수행 후 결과 값 동일 여부 확인
